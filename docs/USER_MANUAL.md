## 1. BOMSyncToolとは

BOMSyncToolは、2つの部品表（BOM）を比較して、どこが違うのかを見つけるためのアプリケーションです。

**こんなときに便利です：**
- 新しい部品表と古い部品表を比べたいとき
- 部品の追加・削除・変更を一目で確認したいとき
- 複数の部品表をまとめて1つにしたいとき
- 部品の型番を社内の表記に統一したいとき

**対応しているファイル形式：**
- CSV（カンマ区切りのテキストファイル）
- Excel（.xlsx）
- CADネットリスト（PADS-ECO、CCF、MSF、PWS、BD、PADS Report）

## 2. インストール方法

### Windowsの場合

1. **インストーラーをダウンロードします**
   - GitHubのリリースページにアクセスします
   - 最新版の「BOMSyncTool_インストーラー.zip」をクリックしてダウンロードします

2. **ZIPファイルを解凍します**
   - ダウンロードしたZIPファイルを右クリックします
   - 「すべて展開」を選択します
   - 「展開」ボタンをクリックします

3. **インストーラーを実行します**
   - 解凍されたフォルダの中にある「USER_MANUAL.pdf」を読みます（このマニュアルのPDF版です）
   - 「BOMSyncTool_セットアップ.exe」をダブルクリックします
   - 「このアプリがデバイスに変更を加えることを許可しますか？」と表示されたら「はい」をクリックします

4. **インストールを進めます**
   - インストーラーの指示に従って「次へ」をクリックします
   - インストール先はそのままで大丈夫です
   - 「インストール」ボタンをクリックします
   - インストールが完了したら「完了」をクリックします

5. **アプリを起動します**
   - デスクトップにできた「BOMSyncTool」のアイコンをダブルクリックします

### macOSの場合

1. **インストーラーをダウンロードします**
   - GitHubのリリースページにアクセスします
   - 最新版の「BOMSyncTool_Mac版.zip」をクリックしてダウンロードします

2. **ZIPファイルを解凍します**
   - ダウンロードしたZIPファイルをダブルクリックします
   - 自動的に解凍されます

3. **アプリをインストールします**
   - 解凍されたフォルダの中にある「USER_MANUAL.pdf」を読みます
   - 「BOMSyncTool.dmg」をダブルクリックします
   - 表示されたウィンドウで、アプリアイコンを「アプリケーション」フォルダにドラッグします

4. **アプリを起動します**
   - Finderで「アプリケーション」フォルダを開きます
   - 「BOMSyncTool」を探してダブルクリックします

> **注意：** 初回起動時に「開発元が未確認」という警告が出る場合があります。その場合は：
> 1. アプリを右クリック（またはControlキー+クリック）します
> 2. 「開く」を選択します
> 3. 再度「開く」をクリックします

│  │                  │      │                  │       │
│  │ ファイルをここに │      │ ファイルをここに │       │

**画面の各部分：**

| 名前 | 説明 |
|------|------|
| **タブ** | 複数のプロジェクト（作業）を切り替えられます。クリックして切り替えます。 |

| **置き換えるボタン** | Dataset AをDataset Bの内容で更新するときに押します。 |
| **結果表示エリア** | 比較した結果が表示される場所です。 |

## 4. 基本的な使い方

### 4.1 はじめてのBOM比較

**手順1：ファイルを読み込む**

1. **ファイルを用意します**
   - 比較したい2つのファイル（例：古い部品表と新しい部品表）を用意します
   - CSVファイルまたはExcelファイルが使えます

2. **Dataset Aにファイルを入れます**
   - 古い部品表のファイルを探します
   - ファイルをクリックしたまま、「Dataset A」と書かれた四角にドラッグします
   - マウスを離すと、ファイルが読み込まれます

   > **ヒント：** ドラッグ＆ドロップが難しい場合は：
   > 1. 「Dataset A」の中にある「ファイルを選択」ボタンをクリック
   > 2. ファイル選択画面が開くので、ファイルを選んでクリック
   > 3. 「開く」ボタンをクリック

3. **Dataset Bにファイルを入れます**
   - 同じように、新しい部品表を「Dataset B」に入れます

**手順2：列の役割を確認する**

ファイルを読み込むと、自動的にどの列が何を表すかを判断します：
- **Ref（部品番号）**：R1、C1、IC1のような部品の番号
- **Part_No（部品型番）**：ABC-1234のような型番
- **Manufacturer（メーカー名）**：会社の名前

もし間違っていたら：
1. 列の一番上にある小さな▼印をクリックします
2. 正しい役割を選びます

**手順3：比較を実行する**

1. 画面の真ん中にある「比較する」ボタンをクリックします
2. 少し待ちます（数秒〜数十秒）
3. 比較結果が表示されます

**手順4：結果を確認する**

結果は色で区別されます：

| 色 | 意味 |
|---|------|
| 🟢 **緑色** | 追加された部品（新しい部品表だけにある） |
| 🔴 **赤色** | 削除された部品（古い部品表だけにある） |
| 🟡 **黄色** | 変更された部品（両方にあるが内容が違う） |
| ⚪ **白色** | 一致している部品（変更なし） |

**手順5：結果を保存する**

1. 画面の右上にある「エクスポート」ボタンをクリックします
2. 保存したい形式を選びます：
   - **CSV**：Excelで開けるファイル（おすすめ）
   - **Excel**：そのままExcelファイル
   - **CAD形式**：設計ツール用（専門的な用途）
3. 保存する場所を選びます（デスクトップがおすすめ）
4. ファイル名を入力します（例：比較結果_2025-11-10）
5. 「保存」ボタンをクリックします

### 4.2 部品表の置き換え

古い部品表を新しい部品表の内容で更新したい場合：

1. **両方のファイルを読み込みます**
   - Dataset Aに古い部品表
   - Dataset Bに新しい部品表

2. **置き換えを実行します**
   - 「置き換える」ボタンをクリックします
   - 少し待ちます

3. **結果を確認します**
   - 置き換え後の内容が表示されます

4. **結果を保存します**
   - 「エクスポート」ボタンをクリックします
   - 保存場所とファイル名を決めて保存します

---

### 4.3 辞書機能の使い方

部品の型番を社内の表記に統一したい場合に使います。

**例：** `ABC-1234` を `抵抗10kΩ` に置き換える

**辞書の登録方法：**

1. **設定画面を開きます**
   - 画面右上の「⚙️設定」ボタンをクリックします

2. **辞書タブを選びます**
   - 「辞書」と書かれたタブをクリックします

3. **登録名リストを開きます**
   - 「登録名リスト」タブをクリックします

4. **新しい辞書を追加します**
   - 「追加」ボタンをクリックします
   - **部品型番**の欄に元の型番を入力します（例：ABC-1234）
   - **登録名**の欄に置き換え後の名前を入力します（例：抵抗10kΩ）

5. **保存します**
   - 入力が終わったらEnterキーを押すか、別の場所をクリックします
   - 自動的に保存されます

**辞書を適用する方法：**

1. **BOMを読み込みます**
   - 通常通りファイルを読み込みます

2. **辞書を適用します**
   - 「辞書」タブにある「BOMに適用」ボタンをクリックします
   - 適用先（Dataset AまたはB）を選びます

3. **結果を確認します**
   - 自動的に型番が置き換わります

**Excelで辞書を作って読み込む方法：**

1. **Excelで辞書ファイルを作ります**
   - Excelを開きます
   - 1行目：`部品型番,登録名` と入力します
   - 2行目以降：型番と登録名を入力します
   - 例：
     ```
     部品型番,登録名
     ABC-1234,抵抗10kΩ
     DEF-5678,コンデンサ100μF
     ```

2. **CSVで保存します**
   - 「ファイル」→「名前を付けて保存」
   - ファイルの種類で「CSV UTF-8」を選びます
   - 保存します

3. **BOMSyncToolに読み込みます**
   - 「辞書」タブを開きます
   - 「CSVインポート」ボタンをクリックします
   - 作成したCSVファイルを選んで「開く」をクリックします

### 4.4 前処理機能

BOMをきれいに整形したい場合に使います。

**使える機能：**
- **Reference展開**：「R1-R4」を「R1」「R2」「R3」「R4」に分ける
- **空欄補完**：空いているマスを自動的に埋める
- **テキストクリーニング**：余分なスペースや記号を削除する

**使い方：**

1. **BOMを読み込みます**
   - 通常通りファイルを読み込みます

2. **編集画面を開きます**
   - 「編集」ボタンをクリックします

3. **前処理を選びます**
   - 「前処理」タブをクリックします
   - 使いたい処理にチェックを入れます

4. **結果をプレビューします**
   - 「適用」ボタンをクリックします
   - 結果が画面に表示されます

5. **問題なければ保存します**
   - 「保存」ボタンをクリックします

### 4.5 プロジェクト管理

複数の仕事を同時に扱う場合に便利です。

**新しいプロジェクトを作る：**
- 画面上部の「[+]」ボタンをクリックします

**プロジェクトを切り替える：**
- 画面上部のタブをクリックして切り替えます

**プロジェクトの名前を変更する：**
1. タブを右クリックします
2. 「名前を変更」を選びます
3. 新しい名前を入力します
4. Enterキーを押します

**よく使うプロジェクトをお気に入りに：**
- タブの横の「⭐」マークをクリックします
- 金色の星になります

**プロジェクトを削除する：**
1. タブの右端にある「×」ボタンをクリックします
2. 確認画面が出たら「はい」をクリックします

## 5. よくある質問

### Q1. ファイルが読み込めません

**A.** 以下を試してください：

1. **ファイルが正しい形式か確認します**
   - CSV（.csv）またはExcel（.xlsx）のファイルですか？
   - 古いExcel形式（.xls）の場合は、新しい形式（.xlsx）で保存し直してください

2. **ファイルが壊れていないか確認します**
   - Excelでファイルを開けますか？
   - 開けない場合はファイルが壊れています

3. **文字コードを確認します（CSVの場合）**
   - Excelで開きます
   - 「ファイル」→「名前を付けて保存」
   - 「ファイルの種類」で「CSV UTF-8 (コンマ区切り)」を選びます
   - 保存します
   - このファイルを読み込み直します

### Q2. 列の役割が自動で認識されません

**A.** 手動で設定してください：

1. 列の一番上にある▼印をクリックします
2. 正しい役割を選びます：
   - **Ref**：部品番号（R1、C1など）
   - **Part_No**：部品型番
   - **Manufacturer**：メーカー名
   - **Ignore**：使わない列
3. プロジェクトを保存すると、次回から自動で設定されます

### Q3. 比較結果が表示されません

**A.** 以下を確認してください：

1. Dataset AとBの両方にファイルが入っていますか？
2. 列の役割（RefとPart_No）が正しく設定されていますか？
3. ファイルの中にデータが入っていますか？（空のファイルではありませんか？）

### Q4. エクスポートしたファイルが開けません

**A.** 保存した形式に合ったソフトで開いてください：

- **CSVファイル**：Excel、メモ帳
- **Excelファイル**：Microsoft Excel
- **CAD形式**：専用の設計ツール

### Q5. 辞書を適用したら値が消えました

**A.** 以下の方法で元に戻せます：

1. **方法1：ファイルを読み込み直す**
   - Dataset Aまたは Bに元のファイルをドラッグ＆ドロップし直します

2. **方法2：前のタブに戻る**
   - まだ辞書を適用していないタブがあれば、そのタブに切り替えます

### Q6. 印刷したいのですが

**A.** 以下の手順で印刷できます：

1. 比較結果を表示します
2. 画面右上の「印刷」ボタンをクリックします
3. 印刷画面が開きます
4. プリンターを選んで「印刷」をクリックします

**PDFで保存したい場合：**
1. 印刷画面で「送信先」または「プリンター」を選びます
2. 「PDFに保存」または「Microsoft Print to PDF」を選びます
3. 「印刷」をクリックします
4. 保存場所とファイル名を決めて保存します

## 6. 困ったときは

### エラーメッセージが表示された場合

エラーメッセージが表示されたら：

1. **メッセージをメモします**
   - エラーメッセージを紙にメモするか、スマートフォンで写真を撮ります

2. **アプリを再起動します**
   - アプリを一度閉じます
   - もう一度アプリを起動します

3. **それでも解決しない場合は報告します**
   - 下の「問題を報告する方法」を見てください

### 問題を報告する方法

バグや不具合を見つけた場合は、GitHubで報告してください。報告していただくことで、問題が解決され、他のユーザーも助かります。

**報告の手順：**

1. **GitHubにアクセスします**
   - インターネットブラウザ（EdgeやChromeなど）を開きます
   - 以下のURLを入力します：
     ```
     https://github.com/[リポジトリのURL]/issues
     ```
   - ※実際のURLは開発者から提供されます

2. **GitHubアカウントを作ります（初回のみ）**
   - GitHubのページ右上の「Sign up」をクリックします
   - メールアドレス、パスワード、ユーザー名を入力します
   - メールで届く確認コードを入力します
   - 「Sign in」でログインします

3. **新しい問題を報告します**
   - 緑色の「New Issue」ボタンをクリックします

4. **問題の内容を書きます**
   - 以下の情報を書いてください：

   **タイトル（Title）：** 問題を一言で（例：CSVファイルが読み込めない）

   **内容（コメント）：**
   ```
   ## 何をしようとしましたか？
   （例：CSVファイルを読み込もうとしました）

   ## 何が起きましたか？
   （例：「ファイルを読み込めません」というエラーが表示されました）

   ## エラーメッセージ
   （表示されたメッセージをそのまま書いてください）

   ## 使用環境
   - OS：（Windows 10、macOS 14など）
   - BOMSyncToolのバージョン：（画面左上に表示されています）
   - ファイルの種類：（CSV、Excelなど）

   ## その他
   （気づいたことがあれば書いてください）
   ```

5. **送信します**
   - 緑色の「Submit new issue」ボタンをクリックします

**注意：** 会社の機密情報を含むファイルやデータは送らないでください。

## 付録：用語集

| 用語 | 説明 |
|------|------|
| **BOM** | Bill of Materials（部品表）の略。製品に使う部品のリスト。 |
| **CSV** | カンマで区切られたテキストファイル。Excelで開けます。 |
| **Dataset** | データセット。比較する部品表のこと。 |
| **Ref** | Reference（部品番号）の略。R1、C1のような番号。 |
| **Part_No** | Part Number（部品型番）の略。ABC-1234のような型番。 |
| **エクスポート** | ファイルとして保存すること。 |
| **インポート** | ファイルを読み込むこと。 |
| **辞書** | 型番を別の名前に置き換えるための対応表。 |
| **プロジェクト** | 1つの作業単位。タブで表示されます。 |

## 付録：トラブル対処早見表

| 困ったこと | 確認すること | 対処方法 |
|-----------|------------|---------|
| ファイルが読み込めない | ファイル形式 | CSVまたはExcel形式で保存し直す |
| エラーが出る | - | アプリを再起動する |
| 列がずれている | 列の役割 | 手動で列の役割を設定する |
| 結果が表示されない | 両方のファイル | 両方のファイルが読み込まれているか確認 |
| 保存したファイルが開けない | ファイル形式 | 正しいソフト（ExcelやCADツール）で開く |
| 動作が遅い | データ量 | 大きなファイルは時間がかかります。そのまま待ってください |
